🔒 IPTables - Estados y Configuración

Aprende a configurar y gestionar reglas de IPTables en un entorno controlado y seguro.

¿Qué es IPTables? 🛡️

IPTables es una herramienta utilizada en sistemas Linux para gestionar reglas de cortafuegos (firewall). Permite definir qué tráfico de red está permitido o bloqueado, garantizando la seguridad del sistema.

🔹 IPTables usa reglas organizadas en tablas y cadenas que controlan el tráfico de red. A continuación, exploraremos cómo listar reglas, establecer políticas y configurar reglas específicas. 🚀

📋 Listar las reglas actuales

Para ver las reglas configuradas en el cortafuegos y entender qué tráfico está permitido o bloqueado, usa el siguiente comando:

sudo iptables -L -n -v

🔹 Este comando mostrará una lista detallada de las reglas activas en el sistema.

⚙️ Establecer políticas predeterminadas

Para mejorar la seguridad, es recomendable definir políticas predeterminadas. Un buen punto de partida es:


sudo iptables -P INPUT DROP  # Bloquea tráfico entrante no autorizado
sudo iptables -P FORWARD DROP  # Bloquea tráfico reenviado
sudo iptables -P OUTPUT ACCEPT  # Permite todo el tráfico saliente

📌 Explicación:

  • INPUT DROP: Bloquea conexiones entrantes no especificadas en otras reglas.
  • FORWARD DROP: Bloquea tráfico que se reenvía a otros dispositivos.
  • OUTPUT ACCEPT: Permite que el sistema envíe tráfico sin restricciones.

🎯 Reglas básicas de IPTables

Las reglas de IPTables permiten definir qué tráfico se acepta o bloquea. Aquí tienes algunas reglas esenciales:

1️⃣ Permitir tráfico SSH (Puerto 22)

Para permitir conexiones SSH en el puerto 22 y evitar bloqueos accidentales:

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

2️⃣ Permitir tráfico desde una IP específica

Si quieres aceptar tráfico solo de una dirección IP concreta, como 192.168.1.100:

sudo iptables -A INPUT -s 192.168.1.100 -j ACCEPT

3️⃣ Bloquear tráfico hacia una IP específica

Para denegar el tráfico de salida a una IP concreta, por ejemplo, 203.0.113.1:

sudo iptables -A OUTPUT -d 203.0.113.1 -j REJECT

4️⃣ Limitar intentos de conexión SSH (Evitar fuerza bruta) 🔐

Para evitar ataques por fuerza bruta en SSH, limita el número de intentos por IP:


sudo iptables -A INPUT -p tcp --dport 22 -m recent --set --name SSH
sudo iptables -A INPUT -p tcp --dport 22 -m recent --update --seconds 60 --hitcount 4 --name SSH -j DROP

5️⃣ Permitir tráfico web (HTTP y HTTPS)

Si el servidor ejecuta una página web, habilita los puertos 80 y 443:


sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

💾 Guardar reglas de IPTables

Por defecto, las reglas de IPTables desaparecen tras reiniciar el sistema. Para hacerlas persistentes en Debian/Ubuntu:

sudo apt install iptables-persistent

Para guardar las reglas manualmente:

sudo iptables-save > /etc/iptables/rules.v4

Y para restaurarlas en cada reinicio:

sudo iptables-restore < /etc/iptables/rules.v4

✅ Buenas prácticas con IPTables

  • 🛠️ **Define reglas específicas antes de las generales** para evitar bloqueos inesperados.
  • 📜 **Registra el tráfico sospechoso antes de bloquearlo**, usando `LOG`:
    • iptables -A INPUT -p tcp --dport 23 -j LOG --log-prefix "Telnet bloqueado: "
  • 📌 **Usa listas blancas en lugar de listas negras**, permitiendo solo tráfico necesario.
  • 🚀 **Testea las reglas antes de aplicarlas permanentemente**.

🎯 Conclusión

IPTables es una herramienta fundamental en la seguridad de redes Linux. Con estas configuraciones básicas, ya puedes gestionar tráfico de manera segura. ¡Practica y mejora tus reglas! 🔥

📚 Glosario Interactivo

DNAT se utiliza para redirigir paquetes a una dirección IP y puerto específicos. Es común en servidores que reciben tráfico desde Internet.

    # Ejemplo de DNAT
    iptables -t nat -A PREROUTING -p tcp --dport 80 -j DNAT --to-destination 192.168.1.10:80

SNAT modifica la dirección IP de origen de los paquetes antes de enviarlos a Internet. Se usa comúnmente en firewalls para permitir que las redes internas accedan a Internet.

    # Ejemplo de SNAT
    iptables -t nat -A POSTROUTING -o eth0 -j SNAT --to-source 203.0.113.1

Stateful Firewall: Mantiene un registro de las conexiones activas y permite el tráfico relacionado. IPTables es un firewall stateful.

Stateless Firewall: No mantiene un registro de conexiones. Cada paquete se evalúa de forma independiente.

  • Escaneo de Puertos: Bloquear intentos de escaneo con reglas específicas.
  • DDoS: Limitar el número de conexiones por IP.
  • IP Spoofing: Filtrar paquetes con direcciones IP falsificadas.


Reglas:

Reglas de IPTables

Las reglas de IPTables se organizan en cadenas (chains) que definen cómo se maneja el tráfico. Las cadenas principales son:

  • INPUT: Para tráfico entrante al servidor.
  • OUTPUT: Para tráfico saliente del servidor.
  • FORWARD: Para tráfico que se reenvía a través del servidor.

Las reglas se aplican en orden, y pueden realizar las siguientes acciones:

  • ACCEPT: Permitir el tráfico.
  • DROP: Descartar el tráfico sin notificar al remitente.
  • REJECT: Rechazar el tráfico y notificar al remitente.
  • LOG: Registrar el tráfico en los logs.

Ejemplo de reglas comunes:

    # Permitir tráfico SSH (puerto 22)
    iptables -A INPUT -p tcp --dport 22 -j ACCEPT

    # Permitir tráfico HTTP (puerto 80) y HTTPS (puerto 443)
    iptables -A INPUT -p tcp --dport 80 -j ACCEPT
    iptables -A INPUT -p tcp --dport 443 -j ACCEPT

    # Bloquear todo el tráfico entrante no permitido
    iptables -A INPUT -j DROP

    # Habilitar NAT para una red interna
    iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
Tipos de Reglas en IPTables

IPTables maneja diferentes reglas organizadas en tablas y cadenas. A continuación, se presentan los principales tipos de reglas:

  • Reglas de Filtrado (filter): Controlan el tráfico permitido o bloqueado.
  • Reglas de NAT (nat): Modifican direcciones IP y puertos en los paquetes.
  • Reglas de Mangle (mangle): Alteran bits en los paquetes para QoS o routing avanzado.
  • Reglas de Raw (raw): Excluyen paquetes del seguimiento de conexiones.
  • Reglas de Seguridad (security): Aplican políticas de seguridad reforzada.
Ejemplo de Reglas:
    # Reglas de filtrado
    iptables -A INPUT -p tcp --dport 80 -j ACCEPT
    iptables -A FORWARD -p tcp --dport 22 -j DROP

    # Reglas de NAT
    iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

    # Reglas de Mangle
    iptables -t mangle -A PREROUTING -p tcp --dport 443 -j TOS --set-tos 0x10
Estados en IPTables
  • NEW: Indica que el paquete es el inicio de una nueva conexión.
  • ESTABLISHED: Paquete pertenece a una conexión ya establecida.
  • RELATED: Conexión relacionada con otra ya existente.
  • INVALID: Paquete desconocido o mal formado.
  • UNTRACKED: Excluido del seguimiento de conexiones.


Ejercicios Prácticos con Soluciones en IPTables

Estos ejercicios están diseñados para aplicarse en las máquinas virtuales creadas, especialmente en Endian_UTM (el firewall) y Public_Web (el servidor en la DMZ).

Objetivo: Configurar reglas básicas de IPTables para permitir tráfico SSH y bloquear el resto.

            # Permitir tráfico SSH (puerto 22) desde la red GREEN (192.168.1.0/24)
            iptables -A INPUT -p tcp --dport 22 -s 192.168.1.0/24 -j ACCEPT
            
            # Permitir tráfico ESTABLISHED y RELATED
            iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
            
            # Bloquear todo el tráfico entrante no permitido
            iptables -A INPUT -j DROP
            
            # Guardar las reglas
            iptables-save > /etc/iptables/rules.v4

Objetivo: Configurar IPTables para permitir tráfico web en el servidor Public_Web.

            # Permitir tráfico HTTP (puerto 80) y HTTPS (puerto 443)
            iptables -A INPUT -p tcp --dport 80 -j ACCEPT
            iptables -A INPUT -p tcp --dport 443 -j ACCEPT
            
            # Permitir tráfico ESTABLISHED y RELATED
            iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
            
            # Bloquear todo el tráfico entrante no permitido
            iptables -A INPUT -j DROP
            
            # Guardar las reglas
            iptables-save > /etc/iptables/rules.v4

Objetivo: Permitir que las máquinas en la red GREEN accedan a Internet.

            # Habilitar el forwarding de paquetes
            echo 1 > /proc/sys/net/ipv4/ip_forward
            
            # Configurar NAT para la red GREEN
            iptables -t nat -A POSTROUTING -o eth3 -j MASQUERADE
            
            # Permitir tráfico desde GREEN a WAN
            iptables -A FORWARD -i eth1 -o eth3 -j ACCEPT
            
            # Guardar las reglas
            iptables-save > /etc/iptables/rules.v4

Objetivo: Configurar IPTables para bloquear tráfico ICMP en PC1_LAN.

            # Bloquear tráfico ICMP (ping)
            iptables -A INPUT -p icmp --icmp-type echo-request -j DROP
            
            # Permitir tráfico ESTABLISHED y RELATED
            iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
            
            # Guardar las reglas
            iptables-save > /etc/iptables/rules.v4

Objetivo: Configurar IPTables para registrar tráfico en Security_Onion.

            # Registrar tráfico entrante
            iptables -A INPUT -j LOG --log-prefix "IPTables-INPUT: "
            
            # Registrar tráfico saliente
            iptables -A OUTPUT -j LOG --log-prefix "IPTables-OUTPUT: "
            
            # Guardar las reglas
            iptables-save > /etc/iptables/rules.v4

Objetivo: Bloquear el acceso a un sitio web específico (por ejemplo, example.com) desde PC1_LAN.

# Bloquear tráfico hacia example.com
iptables -A FORWARD -p tcp --dport 80 -m string --string "example.com" --algo bm -j DROP
iptables -A FORWARD -p tcp --dport 443 -m string --string "example.com" --algo bm -j DROP

# Guardar las reglas
iptables-save > /etc/iptables/rules.v4

Objetivo: Limitar el número de conexiones simultáneas desde una misma IP en Public_Web.

# Limitar a 10 conexiones simultáneas por IP
iptables -A INPUT -p tcp --dport 80 -m connlimit --connlimit-above 10 -j DROP

# Guardar las reglas
iptables-save > /etc/iptables/rules.v4

Objetivo: Redirigir el tráfico del puerto 8080 al puerto 80 en Public_Web.

# Redirigir tráfico del puerto 8080 al puerto 80
iptables -t nat -A PREROUTING -p tcp --dport 8080 -j REDIRECT --to-port 80

# Guardar las reglas
iptables-save > /etc/iptables/rules.v4

Objetivo: Bloquear el acceso desde una IP específica (por ejemplo, 192.168.1.100) a Endian_UTM.

# Bloquear tráfico desde 192.168.1.100
iptables -A INPUT -s 192.168.1.100 -j DROP

# Guardar las reglas
iptables-save > /etc/iptables/rules.v4

Objetivo: Permitir el acceso solo desde IPs específicas a Security_Onion.

# Permitir tráfico solo desde 192.168.1.0/24
iptables -A INPUT -s 192.168.1.0/24 -j ACCEPT

# Bloquear todo el tráfico entrante no permitido
iptables -A INPUT -j DROP

# Guardar las reglas
iptables-save > /etc/iptables/rules.v4

💡 UFW (Uncomplicated Firewall) en vez de IPTables

UFW es una interfaz más sencilla para administrar IPTables. Aquí hay ejemplos de equivalencias entre IPTables y UFW:

# En IPTables:
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# En UFW:
ufw allow 22/tcp

Objetivo: Mostrar que UFW facilita la configuración sin perder flexibilidad.

UFW (Uncomplicated Firewall)

Para una gestión más sencilla del cortafuegos, especialmente para usuarios que prefieren una interfaz más amigable, se recomienda utilizar UFW (Uncomplicated Firewall), que simplifica la configuración de IPTables. UFW proporciona una interfaz de línea de comandos fácil de usar y también cuenta con una interfaz gráfica llamada Gufw.

Instalación y uso básico de UFW

Instalar UFW

En la mayoría de las distribuciones basadas en Debian, UFW viene preinstalado. Si no es así, puede instalarlo con:

sudo apt-get install ufw

Habilitar UFW

Antes de habilitarlo, asegúrese de permitir las conexiones SSH para no perder acceso remoto:

sudo ufw allow ssh
sudo ufw enable

Permitir o denegar servicios

Para permitir el tráfico HTTP y HTTPS:

sudo ufw allow http
sudo ufw allow https

Para denegar un puerto específico, por ejemplo, el 25 (SMTP):

sudo ufw deny 25

Ver el estado y las reglas

Para comprobar el estado de UFW y las reglas activas:

sudo ufw status verbose

Para obtener más información sobre la configuración de UFW en Ubuntu, puede consultar la documentación oficial de Ubuntu.

Interfaz gráfica (Gufw)

Además, si prefiere una interfaz gráfica para gestionar UFW, puede instalar Gufw, que facilita la configuración del cortafuegos sin necesidad de utilizar la línea de comandos.

nftables Sucesor de iptables que proporciona un marco unificado para la filtración y clasificación de paquetes en Linux. firewalld Herramienta de gestión de firewalls que actúa como una interfaz para el marco netfilter del kernel de Linux. Utiliza nftables como backend predeterminado y permite la administración de zonas de firewall con diferentes niveles de confianza. Ofrece una interfaz de línea de comandos y una interfaz gráfica opcional para facilitar la configuración. Wikipedia
Shorewall Herramienta de configuración de firewalls que simplifica la creación de reglas complejas mediante el uso de archivos de configuración en formato de texto plano. Es adecuada para usuarios que prefieren una configuración basada en archivos y buscan una mayor flexibilidad en la gestión de reglas de firewall. Wikipedia

Es especialmente popular en distribuciones como Ubuntu y Debian. Aunque no ofrece todas las funcionalidades avanzadas de iptables o nftables, es una opción adecuada para usuarios que requieren una configuración sencilla y rápida de su firewall. Wikipedia

📜 Análisis de Logs de IPTables

Los logs de IPTables se registran en /var/log/syslog. Aquí hay ejemplos de cómo analizarlos:

# Ver logs en tiempo real
tail -f /var/log/syslog | grep IPTables

# Ejemplo de log:
# Nov  1 12:34:56 server kernel: [IPTables-INPUT] IN=eth0 OUT= MAC=00:11:22:33:44:55 SRC=192.168.1.100 DST=192.168.1.1 LEN=60 TOS=0x00 PREC=0x00 TTL=64 ID=12345 DF PROTO=TCP SPT=54321 DPT=22 WINDOW=64240 RES=0x00 SYN URGP=0

Información Relevante:

  • SRC: Dirección IP de origen.
  • DST: Dirección IP de destino.
  • PROTO: Protocolo (TCP, UDP, ICMP).
  • SPT/DPT: Puerto de origen/destino.
