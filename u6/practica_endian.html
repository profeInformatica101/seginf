Práctica: Configuración de Endian con Zona Verde y Naranja

Descripción

El objetivo de esta práctica es configurar un firewall Endian con una estructura de red segmentada en tres zonas: Zona Verde (red interna), Zona Naranja (DMZ) y Zona Roja (Internet). Los alumnos deberán documentar todo el proceso en un manual técnico con capturas de pantalla y explicaciones detalladas.

Objetivos

Es fundamental asegurarse de que las interfaces de las máquinas virtuales estén correctamente configuradas para garantizar la conectividad y el correcto funcionamiento del firewall.

  • Instalar y configurar Endian en una máquina virtual o física.
  • Configurar las interfaces de red: Zona Verde (LAN), Zona Naranja (DMZ) y Zona Roja (Internet).
  • Establecer reglas de firewall adecuadas para garantizar la seguridad.
  • Implementar un servidor Apache básico en la DMZ y probar la conectividad.
  • Configurar Endian Firewall para permitir tráfico HTTP/HTTPS.
  • Configurar Port Forwarding (DNAT) en Endian para acceso externo.
  • Configurar NAT para permitir que una IP de la Zona Roja redireccione a la DMZ.
  • Documentar el proceso en un manual detallado.

Configuración del Servidor Apache en la DMZ

El servidor Apache deberá estar instalado en la DMZ y configurado para servir una página de prueba accesible desde la Zona Verde pero protegida de accesos externos no autorizados.

  1. Instalar Apache en la máquina de la DMZ: sudo apt update && sudo apt install apache2 -y
  2. Asegurar que el servicio está activo: sudo systemctl start apache2
  3. Verificar la accesibilidad mediante el navegador: http://IP-DMZ
  4. Configurar reglas de firewall en Endian para permitir tráfico solo desde la Zona Verde.

Configuración de Port Forwarding (DNAT) en Endian

Para permitir acceso externo a servicios HTTP/HTTPS en la DMZ, se debe configurar DNAT en Endian:

  1. Acceder a la interfaz web de Endian.
  2. Navegar a la sección Firewall → Port Forwarding.
  3. Agregar una nueva regla con los siguientes parámetros:
    • Protocolo: TCP
    • Puertos de origen: 80 (HTTP), 443 (HTTPS)
    • IP de destino: Dirección IP del servidor en la DMZ
    • Puerto de destino: 80, 443
  4. Guardar los cambios y aplicar las reglas.
  5. Probar la conectividad desde una red externa.

Configuración de NAT en Endian para la Zona Roja

Para permitir que una IP de la Zona Roja (Internet) pueda redirigir tráfico hacia la DMZ, se deben seguir los siguientes pasos:

  1. Acceder a la interfaz web de Endian.
  2. Ir a Firewall → NAT.
  3. Crear una nueva regla de NAT con los siguientes parámetros:
    • Protocolo: TCP
    • IP de origen: Dirección IP específica de la Zona Roja
    • IP de destino: Dirección IP del servidor en la DMZ
    • Puertos: 80, 443
  4. Aplicar la configuración y realizar pruebas de conectividad.
